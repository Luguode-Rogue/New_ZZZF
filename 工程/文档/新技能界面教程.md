# New_ZZZF 新技能界面教程

> 本文档描述新版本技能界面的使用流程。如果你正在使用旧版教程（基于InventoryLogic体系的SkillInventoryManager流程），请重点阅读**第一章**，了解需要改动的地方。

---

## 一、重要变化概览（对比旧教程）

### 1.1 需要改动的地方

| 项目 | 旧教程（已废弃） | 新教程（当前） |
|------|-------------------|----------------|
| **入口方法** | `SkillInventoryManager.OpenInventoryPresentation()` | `ScreenManager.PushScreen(new CustomSkillScreen())` |
| **界面Screen类** | `GauntletSkillScreen` | `CustomSkillScreen` |
| **ViewModel** | `SPSkillVM` | `CustomSkillScreenVM` |
| **界面XML** | 游戏内置 `Inventory` | 自定义 `CustomSkillScreen.xml` |
| **游戏状态** | `SkillInventoryState` → `InventoryLogic` | 无（直接Push Screen，不依赖GameState） |
| **技能选择** | 无专用弹窗 | 新增 `SkillSelectionPopup.xml` + `SkillSelectionVM` |
| **技能槽位** | 未知 | 固定8槽：`MainActive`/`SubActive`/`Passive`/`CombatArt`/`Spell0`~`Spell3` |

### 1.2 不需要改动的地方

以下内容**完全不变**，继续沿用旧教程的流程：

1. **新增技能的流程**（继承`SkillBase` → 在`SkillFactory._skillRegistry`注册 → 添加翻译）
2. **新增buff状态的流程**（继承`AgentBuff` → 修改`ZZZF_SandboxAgentStatCalculateModel`等）
3. **士兵技能配置**（`troop_skills.xml`格式不变）

## 二、新技能界面架构

### 2.1 核心流程

```
用户打开界面
  → ScreenManager.PushScreen(new CustomSkillScreen())
  → CustomSkillScreen.OnInitialize()
    → 创建 CustomSkillScreenVM()
      → SkillCatalog.LoadFromFactory()     // 从SkillFactory._skillRegistry加载所有技能
      → CreateSkillSlots()                  // 创建8个技能槽位ViewModel
      → PopulateRoster()                   // 从Clan.PlayerClan读取英雄列表
    → 创建 GauntletLayer
    → LoadMovie("CustomSkillScreen", _dataSource)  // 加载自定义XML
```

### 2.2 文件清单

| # | 文件 | 路径 | 职责 |
|---|------|------|------|
| 1 | `CustomSkillScreen.cs` | `New_ZZZF/工程/New_ZZZF/GUI/` | Screen层：GauntletLayer管理、Movie加载/释放、键盘输入 |
| 2 | `CustomSkillScreenVM.cs` | 同上 | 主ViewModel：英雄列表、技能槽、英雄切换、应用/撤销 |
| 3 | `SkillSelectionVM.cs` | 同上 | 弹窗ViewModel：技能列表、搜索过滤、选择回调 |
| 4 | `SkillItemVM.cs` | 同上 | 弹窗中的单个技能项ViewModel |
| 5 | `SkillModel.cs` | 同上 | 数据模型：SkillUIData、HeroSkillData、SkillCatalog |
| 6 | `SkillConfigManager.cs` | `New_ZZZF/工程/New_ZZZF/Systems/` | 技能配置持久化：XML加载/保存 |
| 7 | `CustomSkillScreen.xml` | `New_ZZZF/GUI/Prefabs/` | 主界面布局 |
| 8 | `SkillSelectionPopup.xml` | 同上 | 技能选择弹窗布局 |

### 2.3 类继承关系

```
ScreenBase
  └── CustomSkillScreen           ← 入口Screen
        ├── GauntletLayer _gauntletLayer
        ├── GauntletMovieIdentifier _movie (主界面)
        ├── GauntletMovieIdentifier _popupMovie (弹窗)
        └── CustomSkillScreenVM _dataSource

ViewModel (TaleWorlds.Library)
  ├── HeroVM                      ← 左侧英雄列表项
  ├── SkillSlotVM                 ← 右侧技能槽位
  ├── CustomSkillScreenVM         ← 主界面ViewModel
  │     ├── Roster (英雄列表)
  │     ├── Skills (技能槽列表)
  │     └── SkillSelectionPopup (弹窗ViewModel引用)
  ├── SkillSelectionVM            ← 弹窗ViewModel
  └── SkillItemVM                ← 弹窗中的技能项

数据模型（非ViewModel）
  ├── SkillUIData               ← UI层技能展示数据
  ├── HeroSkillData              ← 英雄技能装备配置
  ├── SkillCatalog              ← 全技能目录
  └── SkillConfigManager        ← 持久化单例
```

## 三、使用新技能界面的完整流程

### 3.1 打开技能界面

**旧教程（已废弃）**：
```csharp
// 旧入口 - 不再使用
SkillInventoryManager.OpenInventoryPresentation()
```

**新教程（当前）**：
```csharp
// 新入口 - 直接Push Screen
ScreenManager.PushScreen(new CustomSkillScreen());
```

### 3.2 界面操作指南

打开界面后，操作流程如下：

1. **选择英雄**：左侧列表点击英雄名称，或按 `↑` `↓` 键导航
2. **选择技能槽位**：点击右侧技能卡片，或按 `1`~`8` 快捷键
3. **选择技能**：在弹窗中点击技能，或用 `↑` `↓` 导航 + `Enter` 确认
4. **搜索技能**：在弹窗搜索框输入文字，实时过滤技能列表
5. **应用更改**：点击"应用更改"按钮，或按 `Ctrl+S`
6. **撤销更改**：点击"撤销"按钮，或按 `Ctrl+Z`
7. **关闭界面**：按 `ESC`

### 3.3 技能槽位说明

| 槽位ID | 显示标签 | 接受的技能类型 |
|---------|-----------|----------------|
| `MainActive` | 主主动 | `SPSkillType.MainActive` |
| `SubActive` | 副主动 | `SPSkillType.SubActive` |
| `Passive` | 被动 | `SPSkillType.Passive` |
| `CombatArt` | 战技 | `SPSkillType.CombatArt` |
| `Spell0` | 法术① | `SPSkillType.Spell` |
| `Spell1` | 法术② | `SPSkillType.Spell` |
| `Spell2` | 法术③ | `SPSkillType.Spell` |
| `Spell3` | 法术④ | `SPSkillType.Spell` |

## 四、新增技能的流程（不变）

> 这部分内容与旧教程**完全相同**，无需修改。

### 4.1 创建技能类

1. 新建一个类，继承 `SkillBase`
2. 完成构造方法、`Activate` 方法、`CheckCondition` 方法

**示例：嘲讽技能**

```csharp
using New_ZZZF.Systems;
using TaleWorlds.Localization;
using TaleWorlds.Core;
using TaleWorlds.MountAndBlade;

namespace New_ZZZF
{
    internal class ChaoFeng : SkillBase
    {
        public ChaoFeng()
        {
            SkillID = "ChaoFeng";      // 必须唯一
            Type = SkillType.MainActive;    // 类型必须明确
            Cooldown = 2;             // 冷却时间（秒）
            ResourceCost = 0f;        // 消耗
            Text = new TextObject("{=ZZZF0023}嘲讽");
            Description = new TextObject("{=ZZZF0024}嘲讽附近敌方单位...");
        }

        public override bool Activate(Agent agent)
        {
            // 技能效果逻辑
            List<AgentBuff> newStates = new List<AgentBuff> { new ChaoFengBuffApplyToSelf(30f, agent), };
            foreach (var state in newStates)
            {
                state.TargetAgent = agent;
                agent.GetComponent<AgentSkillComponent>().StateContainer.AddState(state);
            }
            return true;
        }

        public override bool CheckCondition(Agent caster)
        {
            // 技能释放条件
            return true;
        }

        // 内部Buff类定义...
    }
}
```

### 4.2 注册技能

在 `SkillFactory` 类的 `_skillRegistry` 字典中添加新技能：

```csharp
public static readonly Dictionary<string, SkillBase> _skillRegistry = new Dictionary<string, SkillBase>
{
    {"NullSkill", new NullSkill() },
    {"JianQi", new JianQi() },          // 剑气斩
    {"ChaoFeng", new ChaoFeng() },     // 嘲讽（新增）
    // ... 其他技能
};
```

### 4.3 添加翻译

在 `ModuleData\languages\CNs\ZZZF_Skill-CN.xml` 中添加对应翻译：


## 五、新增Buff状态的流程（不变）

> 这部分内容与旧教程**完全相同**，无需修改。

### 5.1 创建Buff类

1. 新建一个类，继承 `AgentBuff`
2. 完成构造函数、`OnApply`、`OnUpdate`、`OnRemove` 方法

### 5.2 修改属性强化逻辑

对于强化人物属性（如增加移动速度）的buff，需要在 `ZZZF_SandboxAgentStatCalculateModel` 类的 `UpdateHumanStats` 方法中添加代码。

> **注意**：单纯调用 `agent.AgentDrivenProperties` 修改数值只能在一帧生效，必须使用重写 `UpdateHumanStats` 的方法让它持续生效。

### 5.3 修改伤害强化逻辑

- 百分比增伤：在 `WOW_Script_AgentStatCalculateModel` 中添加代码
- 固定数值增伤：在 `Systems\NewDamageModel.cs` 中调整

### 5.4 附加Buff给士兵的代码示例

```csharp
public override bool Activate(Agent agent)

    foreach (var state in newStates)
    {
        state.TargetAgent = agent;
        agent.GetComponent<AgentSkillComponent>().StateContainer.AddState(state);
    }
    return true;
}
```

## 八、常见问题

### Q: 如何添加/删除技能槽位？

修改 `CustomSkillScreenVM.cs` 中的 `CreateSkillSlots()` 方法（约Line 431-455）。

### Q: 如何修改技能类型的过滤？

修改 `CustomSkillScreenVM.cs` 中的 `slotDefs` 数组（约Line 437-448）。

### Q: 技能保存后重启丢失？

1. 检查 `SkillConfigManager._troopSkillMap` 是否正确加载
2. 检查 `SaveToXml` 的路径和权限
3. 检查 `SetSkillSetForTroop` 逻辑

### Q: 英雄列表为空？

1. 检查 `Clan.PlayerClan` 是否为null
2. 检查过滤条件：存活 + 成年 + Active状态

## 九、附录：完整数据流

### 9.1 技能选择数据流

```
用户点击技能槽
  → SkillSlotVM.ExecuteClick()
    → CustomSkillScreenVM.OnSlotClicked(slotVM)
      → 创建 SkillSelectionVM(catalog, filterType, onSelect, onClose)
      → SkillSelectionPopup = skillSelectionVM  // 触发弹窗显示

CustomSkillScreen.OnFrameTick() 检测到弹窗打开
  → LoadMovie("SkillSelectionPopup", viewModel)

用户选择技能
  → SkillItemVM.ExecuteSelect()
    → SkillSelectionVM.OnSkillSelected(skillData)
      → 回调 onSkillSelected(skillData)
        → CustomSkillScreenVM.AssignSkillToSlot(slotVM, skillData)
          → slotVM.SetSkill(skillData)
          → 同步到 _currentHeroSkillData
          → IsDirty = true
```

### 9.2 数据保存流

```
用户点击"应用更改"
  → CustomSkillScreenVM.ExecuteApply()
    → SaveCurrentHeroSkills()
      → _currentHeroSkillData.Save()
        → SkillConfigManager.Instance.SetSkillSetForTroop()
          → 写入 _troopSkillMap
          → SaveToXml() 持久化到XML文件
```
