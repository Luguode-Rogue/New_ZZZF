# 物品前后缀词缀系统 — 功能文档

**模块路径：** `New_ZZZF\EquipmentAffixSystem`  
**最后更新：** 2026-06-14  
**状态：** 生产可用，UI层+战役层+战斗层已完整接线

## 一、系统概述

### 1.1 设计目标

仿暗黑2装备词缀系统，为骑砍2的物品（武器/护甲/盾牌/马匹）随机附加**前缀**和**后缀**，每个词缀提供属性修正（伤害+、护甲+、吸血、元素伤害等），词缀组合同物品实例绑定的**唯一 ItemModifier**区分显示。

### 1.2 核心概念

| 概念 | 对应类型 | 说明 |
|------|---------|------|
| 词缀定义 | `AffixDefinition` | 静态数据：一条词缀模板（Id、名称、权重、等级范围、属性修正） |
| 词缀实例 | `AffixInstance` | 运行时数据：一件物品具体随到的前后缀组合 + 已结算属性 |
| 物品记录 | `AffixedItemRecord` | 物品实例包装：InstanceId + BaseItemId + Affix + 来源 + 堆叠数 |
| 装备绑定 | `AffixBinding` | 记录物品当前归属于谁（OwnerId + SlotIndex） |
| 修饰符桥接 | `ModifierToInstanceMap` | ItemModifier.StringId → InstanceId，解决UI层无法获取InstanceId的问题 |

### 1.3 稀有度体系

| 稀有度 | 英文 | 颜色 | 前缀数 | 后缀数 | 说明 |
|--------|------|------|--------|--------|------|
| 普通 | Normal | 白色 | 0 | 0 | 无词缀 |
| 魔法 | Magic | 蓝色 | ≤1 | ≤1 | 1-2条词缀 |
| 稀有 | Rare | 黄色 | 1-3 | 1-3 | 2-6条词缀 |
| 暗金 | Unique | 暗金色 | ≤1 | ≤1 | 固定词缀组合 |

## 三、数据模型详解

### 3.1 AffixDefinition（词缀模板）

**12个字段：**

```csharp
public sealed class AffixDefinition
{
    string Id;                      // "affix_cruel" — 唯一标识
    bool IsPrefix;                  // true=前缀, false=后缀
    string DisplayName;             // "残忍的"、"狼之" — 显示用名称
    string Description;             // 描述文本
    int Weight;                     // 抽取权重（越高越常见）
    int MinItemLevel;               // 最低物品等级要求
    int MaxItemLevel;               // 最高物品等级（0=无上限）
    List<string> AllowedItemTypes;  // 允许的装备类型（OneHanded/Bow/BodyArmor...）
    List<string> ConflictAffixIds;  // 互斥词缀ID列表
    Dictionary<string,float> StatModifiers; // 属性修正表
    string Rarity;                  // 品质颜色（Magic/Rare/Unique）
}
```

**`ConflictsWith()` 方法：** 遍历 `ConflictAffixIds`，若已选词缀包含任意互斥ID则返回true，抽选时自动排除。

### 3.2 AffixInstance（词缀实例）

**9个字段（7存档 + 2非存档缓存）：**

```csharp
public class AffixInstance
{
    // ===== 存档键 =====
    string InstanceId;                           // Guid唯一标识
    string BaseItemId;                           // 基础物品StringId
    int ItemLevel;                               // 生成时的物品等级

    // ===== 词缀数据 =====
    List<string> PrefixIds;                      // 前缀ID列表（0-3个）
    List<string> SuffixIds;                      // 后缀ID列表（0-3个）
    Dictionary<string,float> FinalStatModifiers; // 属性修正结算值
    string Rarity;                               // Normal/Magic/Rare/Unique

    // ===== 非存档缓存 =====
    [NonSerialized] List<AffixDefinition> _cachedPrefixDefs;
    [NonSerialized] List<AffixDefinition> _cachedSuffixDefs;
}
```

**核心方法：**

| 方法 | 说明 |
|------|------|
| `HasAnyAffix` | 是否有任何词缀（PrefixIds或SuffixIds非空） |
| `ResolveDefinitions(database)` | 读档后从数据库重建缓存（因缓存不存档） |
| `BuildFullName(baseItemName)` | 构建暗黑2风格完整名称 |
| `GetPrefixDefinitions()` | 获取已解析的前缀定义列表 |
| `GetSuffixDefinitions()` | 获取已解析的后缀定义列表 |
| `StripConnector(name)` | 去掉中文词缀名末尾的"的"或"之" |

### 3.3 AffixedItemRecord（物品实例记录）

```csharp
public sealed class AffixedItemRecord
{
    string InstanceId;     // Guid.NewGuid().ToString("N") — 构造时自动生成
    string BaseItemId;     // ItemObject.StringId
    string Source;         // "PlayerInventory" / "Debug" / "Reroll"
    int StackCount;        // 堆叠数量
    AffixInstance Affix;   // 绑定的词缀实例
    bool IsEquipped;       // 是否已装备到角色身上
}
```

**设计要点：** 每次 `new AffixedItemRecord()` 自动生成唯一 InstanceId，同一底材的不同实例不会冲突。

### 3.4 AffixBinding（绑定记录）

```csharp
public sealed class AffixBinding
{
    string InstanceId;          // 关联的 AffixedItemRecord.InstanceId
    AffixOwnerType OwnerType;   // Inventory / Equipment / Loot / Shop / Debug
    string OwnerId;             // HeroId / PartyId / ShopId
    int SlotIndex;              // 装备槽位索引（EquipmentIndex转int）
}

public enum AffixOwnerType { Inventory, Equipment, Loot, Shop, Debug }
```

**BindingMap 的 Key 格式：** `"OwnerId:SlotIndex"`，如 `"hero_123:4"`

## 四、完整数据流

### 4.1 生成流程（物品获得时）

```
PlayerInventoryExchangeEvent ──→ OnPlayerInventoryExchange()
    │
    ├─ 1. CreateAffixedRecord(item, stackCount, source)
    │   ├─ ClassifyItemType(item)         → "OneHanded" / "BodyArmor" / "Shield" ...
    │   ├─ CalculateItemLevel(item)       → 1~100
    │   ├─ new AffixedItemRecord()         → InstanceId = Guid.NewGuid()
    │   └─ AffixGenerator.GenerateSeeded(baseItemId, itemType, itemLevel, seed)
    │       ├─ AffixDatabase.GetAvailablePrefixes(itemType, itemLevel) → 前缀池
    │       ├─ AffixDatabase.GetAvailableSuffixes(itemType, itemLevel) → 后缀池
    │       ├─ DetermineRarity(itemLevel, roll)
    │       │   ├─ Normal  (Lv1:90% → Lv100:40%)
    │       │   ├─ Magic   (Lv1:8%  → Lv100:48%)
    │       │   ├─ Rare    (Lv1:1%  → Lv100:16%)
    │       │   └─ Unique  (Lv1:0%  → Lv100:1%)
    │       ├─ RollAffixes(pool, maxCount, ...)  → 权重随机 + 冲突排除
    │       └─ ComputeFinalStats(...)             → 属性叠加（相同stat累加）
    │
    ├─ 2. ItemRecordMap[instanceId] = record
    │
    ├─ 3. SyncAffixModifiersToPlayerRoster()
    │   ├─ 遍历玩家背包，找到BaseItemId匹配但无Modifier的元素
    │   ├─ CreateOrGetItemModifier("zzzf_affix_" + InstanceId, InstanceId)
    │   │   ├─ 反射设置 PriceMultiplier = 1.0
    │   │   ├─ Name = "{ITEMNAME}"（必须含此占位符）
    │   │   ├─ MBObjectManager.RegisterPresumedObject(modifier)
    │   │   └─ RegisterModifierBinding(modifierId, instanceId) → ModifierToInstanceMap
    │   └─ roster.Remove(old) + roster.AddToCounts(newElement, amount)
    │
    └─ 4. SyncHeroEquipmentBindings() → BindingMap
```

### 4.2 UI 显示流程（背包/物品详情）

```
背包打开 ──→ ItemVM.get_ItemDescription ──→ 原版返回名称
    │                                         │
    │                                    [HarmonyPostfix]
    │                                    AffixItemDescriptionGetterPatch
    │                                         │
    │                                   GetAffixForEquipmentElement(element)
    │                                         │
    │                             ┌───────────┴───────────┐
    │                        modifierId存在            modifierId为null
    │                       且在ModifierToInstanceMap中   或不在Map中
    │                             │                       │
    │                   ModifierToInstanceMap            return null
    │                     [modifierId] = instId           (保持原版名称)
    │                             │
    │                   ItemRecordMap[instId].Affix
    │                             │
    │                   affix.BuildFullName(baseItemName)
    │                             │
    │                     替换 __result
```

**关键决策：** `GetAffixForEquipmentElement` 不做模板回退。原版 modifier（bent_cheap/well_crafted_cheap）和无 modifier 物品直接返回 null → UI 显示原版名称。只有通过 `ModifierToInstanceMap` 精确匹配的（即 `zzzf_affix_*` modifier）才替换为词缀名。

### 4.3 战斗流程（Mission 层）

```
Mission启动 ──→ AffixMissionBehavior.OnAgentBuild(agent)
    │
    ├─ 获取 HeroObject → 遍历所有装备槽位 (Weapon0~HorseHarness)
    │
    ├─ ResolveInstanceId(behavior, hero, slot, baseItemId)
    │   ├─ 优先级1: BindingMap 精确查找 → behavior.GetEquippedInstanceId(hero, slot)
    │   └─ 优先级2: GetAffixByBaseItemId(baseItemId) 模板回退（过渡方案）
    │       └─ 遍历 ItemRecordMap 找到首个 BaseItemId 匹配的 Record → Affix.InstanceId
    │
    ├─ ctx.SlotToInstanceId[slot] = instanceId
    │
    └─ 存储到 _agentAffixCache[agent.Index] = ctx

战斗内 ──→ NewDamageModel.GetAffixDamageMultiplier(...)
    │
    ├─ AffixMissionBehavior.GetAgentWeaponInstanceId(agent, slot)
    │   └─ _agentAffixCache[agent.Index].GetInstanceId(slot) → instanceId
    │
    └─ 有InstanceId → 从 ItemRecordMap 获取 Affix → 应用属性修正
       无InstanceId → 模板回退 → AffixCampaignBehavior.GetAffixByBaseItemId(baseItemId)
```

**已知限制：** Mission 层模板回退导致同底材多实例在战斗中共享词缀效果。彻底解决需等 Agent 装备槽 → InstanceId 映射完善。

### 4.4 存档/读档流程

```
存档 (SyncData.IsSaving)
    │
    ├─ SerializeRecordMap()
    │   └─ 每行：InstanceId|BaseItemId|ItemLevel|Rarity|Source|StackCount|IsEquipped|prefixIds|suffixIds|statModifiers
    │
    ├─ SerializeBindingMap()
    │   └─ 每行：InstanceId|OwnerType|OwnerId|SlotIndex
    │
    └─ SerializeModifierMap()
        └─ 每行：modifierId|instanceId

读档 (SyncData.IsLoading)
    │
    ├─ DeserializeRecordMap()  → ItemRecordMap
    │   ├─ parts.Length ≥ 10 → v2格式（10字段）
    │   └─ parts.Length < 10  → v1兼容（7字段，Source/StackCount/IsEquipped缺省）
    │
    ├─ DeserializeBindingMap() → BindingMap
    ├─ DeserializeModifierMap() → ModifierToInstanceMap
    │
    ├─ 重建 [NonSerialized] 缓存
    │   └─ foreach record: record.Affix.ResolveDefinitions(db)
    │
    ├─ RebuildModifiers()        → 重建 ItemModifier 对象到 MBObjectManager
    ├─ SyncAffixModifiersToPlayerRoster()  → 修复旧存档中Modifier缺失的物品
    └─ SyncHeroEquipmentBindings()         → 重建装备槽绑定
```

## 五、命名规则（BuildFullName）

```
仅有前缀     → "残忍的 长剑"          (prefixName + " " + baseName)
仅有后缀     → "狼 长剑"              (suffixName + " " + baseName)  
前后缀都有   → "狂战士之狼的长剑"      (prefixStem之 suffixStem的 baseName)
无词缀       → "长剑"                  (baseName原样)
```

**去接头词规则：** `StripConnector()` 去掉前缀/后缀名末尾的"的"或"之"，然后在整个命名中重建"之"和"的"。

**示例：** "锋利的" + "愚人之" + "干草叉" → 去接头词 → "锋利" + "愚人" → "锋利之愚人的干草叉"

## 六、ItemModifier 桥接机制

### 6.1 为什么需要桥接

骑砍2的 `EquipmentElement` 只暴露 `ItemModifier`（原版的概念），不暴露自定义的 `InstanceId`。UI 层（ItemVM）只能拿到 `EquipmentElement`，无法直接获取词缀系统的 `InstanceId`。

### 6.2 桥接方式

| 步骤 | 操作 | 位置 |
|------|------|------|
| 1 | 为每个词缀记录创建唯一的 `ItemModifier` | `CreateOrGetItemModifier()` |
| 2 | modifierId = `"zzzf_affix_" + InstanceId` | 命名约定 |
| 3 | `ModifierToInstanceMap[modifierId] = instanceId` | `RegisterModifierBinding()` |
| 4 | 设置 `PriceMultiplier = 1.0`（反射） | 维持原价 |
| 5 | 设置 `Name = "{ITEMNAME}"` | 必须含此占位符，否则原版 `GetModifiedItemName()` 返回空 |
| 6 | `MBObjectManager.RegisterPresumedObject(modifier)` | 注册才能被序列化和识别 |

### 6.3 查回链路

```
EquipmentElement.ItemModifier.StringId
    → "zzzf_affix_{InstanceId}"            ← 我们的词缀
    OR → "bent_cheap"                      ← 原版modifier（不在Map中）
    OR → "well_crafted_cheap"              ← 原版modifier（不在Map中）
    OR → null                              ← 无modifier

    ↓ ModifierToInstanceMap.TryGetValue(modifierId, out instanceId)

    命中 → ItemRecordMap[instanceId].Affix  ← 返回词缀
    未命中 → return null                     ← 保持原版显示
```

## 八、词缀数据库（AffixDatabase）

### 8.1 初始化

`AffixDatabase.Instance.Initialize()`：
- 硬编码注册全部前缀和后缀定义（Id/名称/权重/属性修正/冲突关系）

- 提供 `GetAvailablePrefixes(itemType, itemLevel)` 和 `GetAvailableSuffixes(itemType, itemLevel)` 查询接口

### 8.2 装备类型分类（ClassifyItemType）

| ItemObject.ItemType | 分类结果 |
|---------------------|---------|
| OneHandedWeapon | "OneHanded" |
| TwoHandedWeapon | "TwoHanded" |
| Bow | "Bow" |
| Crossbow | "Crossbow" |
| Shield | "Shield" |
| HeadArmor | "HeadArmor" |
| BodyArmor | "BodyArmor" |
| LegArmor | "LegArmor" |
| HandArmor | "HandArmor" |
| Cape | "Cape" |
| Horse | "Horse" |
| HorseHarness | "HorseHarness" |

### 8.3 稀有度概率公式

```
normalChance  = max(0,  90 - itemLevel × 0.5)     // Lv1:90% → Lv100:40%
magicChance   = min(40, 8  + itemLevel × 0.4)     // Lv1:8%  → Lv100:48%
rareChance    = min(15, 1  + itemLevel × 0.15)    // Lv1:1%  → Lv100:16%
uniqueChance  = min(2,  itemLevel × 0.01)          // Lv1:0%  → Lv100:1%
```

## 十二、API 约定与禁忌

### ✓ 正确的查询方式

```csharp
// UI层：有 EquipmentElement 时
var affix = behavior.GetAffixForEquipmentElement(element);

// Campaign层：有 InstanceId 时
var affix = behavior.GetAffixByInstanceId(instanceId);

// 生成物品
var record = behavior.CreateAffixedRecord(item, stackCount, source);
```

### ✗ 禁止的做法

```csharp
// ❌ 不要用模板回退来显示物品名称（会污染原版物品）
var affix = behavior.GetAffixByBaseItemId(item.StringId);  // [Obsolete]

// ❌ 不要直接删 ItemRecordMap 条目而不清理 ModifierToInstanceMap
// ❌ 不要用空值/硬编码常量（如 ItemModifier = null）跳过属性修正
// ❌ 不要用 Debug.Print 输出日志（用 InformationManager.DisplayMessage 或 File.AppendAllText）
```

### 注意事项

1. **防递归：** `_isProcessingInventory` 标志防止 `SyncAffixModifiersToPlayerRoster` 的 Remove+Add 触发二次 `PlayerInventoryExchangeEvent`
2. **序列化限制：** Dictionary 不能直接存档，全部编码为 `List<string>`，每行一条记录
3. **非存档缓存：** `_cachedPrefixDefs/_cachedSuffixDefs` 每次读档后需调用 `ResolveDefinitions()` 重建
4. **Modifier 对象：** `ItemModifier` 是通过反射创建的，非原版途径，需通过 `MBObjectManager.RegisterPresumedObject` 注册才能被存档系统识别
