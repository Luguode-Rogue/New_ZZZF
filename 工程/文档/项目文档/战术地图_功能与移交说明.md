# 战术地图（TacticalMap）功能说明与移交文档

> 模块：New_ZZZF ｜ 代码位置：`工程/New_ZZZF/TacticalMap/` ｜ UI 预制体：`GUI/Prefabs/TacticalMap.xml`
> 目标版本：**Bannerlord 1.4.6 运行时**（也兼容编译引用集与运行时 `Rectangle2D` 字段/属性不一致的新版本）
> 文档性质：规范副本，唯一权威说明。下次改动后请同步更新本文。

---

## 0. 30 秒看懂

战术地图是战斗内的 **RTS 风格小地图 HUD 叠加层**，按 **N** 开关，显示在屏幕右上角。它把整张战场地形烘焙成一张底图，并在上面叠加风险区、单位密度、编队/玩家/镜头标记，支持点击下令与镜头联动。设计目标是 **draw call 极低、与单位数量无关**（地形/风险/单位层都是单张烤好的 GPU 纹理整体拉伸），500v500 也不卡。

整个 `TacticalMap/` 文件夹自包含，与 ZZZF 其余系统只在 `SubModule.cs` 的两行调用处耦合，可整体搬成独立 mod。

## 1. 功能清单（当前已有哪些功能）

> 给功能 owner 看：下面每一项都是“现在能用的”。开关都在 `TacticalMap/Config/TacticalSettings.cs`，按 F6 在 `TacticalSettings` 实例里实时改或改默认值均可。

| # | 功能 | 怎么触发 / 在哪看到 | 总开关 | 子开关（`TacticalSettings`） |
|---|------|--------------------|--------|------------------------------|
| 1 | **开关小地图** | 战斗内按 **N** | `EnableMinimap=true` | `ToggleKey=N` |
| 2 | **地形底图** | 小地图底色（高度色带 + 水/林/崖着色） | 同上 | 常开（烘焙纹理） |
| 3 | **风险叠加层** | 崖红 / 水蓝 / 林绿 半透明叠加 | 同上 | `EnableRiskOverlay=true` |
| 4 | **单位密度热力图** | 橙黄半透明点阵，越密越亮（上限 0.3） | 同上 | `EnableDensityHeatmap=true` |
| 5 | **单位层（每 agent 一个点）** | 我方青蓝 / 敌方红 / 中立灰，清晰呈现真实分布 | 同上 | `EnableAgentMarkers=true` |
| 6 | **编队标记** | 每个编队质心一个「队伍色填充方块 + 红(敌)/绿(友)/白(玩家)描边 + 朝向线」 | 同上 | `EnableUnitMarkers=true` |
| 7 | **玩家标记** | 青色空心圆环 + 亮黄中心点（刻意区别于方块/点云） | 同上 | 常开（取 `Controller.PlayerPos`） |
| 8 | **镜头目标指示** | 橙色菱形（对应图例“相机”） | 同上 | 常开（取 `Controller.CameraTarget`） |
| 9 | **点击下令** | 左键=移动到点；**Shift+左键**=攻击移动；右键=朝向某点 | 同上 | 经 `OrderSystem` / `SelectionSystem` |
| 10 | **镜头联动（点击飞镜头）** | 按 **C** 切换模式，开启后点地图镜头平滑飞向该点 | 同上 | `EnableCameraLink=true` / `CameraFollowKey=C` |

### 渲染顺序（后画盖上面）
背景 → 地形纹理 → 风险纹理 → 密度热力 → 单位层纹理 → **编队标记** → **玩家标记** → 镜头目标。

### 操作速查
| 操作 | 行为 | 代码入口 |
|------|------|----------|
| N | 开关小地图 | `TacticalMapMissionLogic.OnMissionTick` → `Controller.SetVisible` |
| C | 切换“镜头联动/跟随”模式 | `Controller.ToggleCameraFollow` |
| 左键点地图 | 移动（保留阵型） | `Controller.HandleClick(shift=false, right=false)` |
| Shift+左键点地图 | 攻击移动 | `Controller.HandleClick(shift=true, right=false)` |
| 右键点地图 | 朝向某点 | `Controller.HandleClick(right=true)` |

> 说明：小地图左侧 `TMapLegend` 文本控件即图例，内容与各标记实际样式一致（玩家=青环+黄心、编队=红/绿/白框、相机=橙菱形）。编队框色按敌我区分：**红=敌人、绿=友军、白=玩家（不变）**；填充仍用队伍色以便区分阵营。

## 2. 快速上手（给下一个智能体）

### 2.1 启动链路（在哪里接线）
```
SubModule.OnSubModuleLoad
  └─ TacticalMapBootstrap.OnSubModuleLoad()        // FeatureGate.Enabled 门控
       └─ new Harmony("TacticalMap") + TacticalCameraPatch.Patch()   // 镜头联动补丁

SubModule.OnMissionBehaviorInitialize(mission)
  └─ TacticalMapBootstrap.OnMissionStart(mission)  // FeatureGate.Enabled 门控
       └─ mission.AddMissionBehavior(new TacticalMapMissionLogic())

TacticalMapMissionLogic (MissionBehavior)
  ├─ OnAfterMissionCreated / 懒初始化兜底(tick) → new TacticalMapController(Mission).Initialize()
  └─ OnMissionTick：处理 N/C/点击；可见时驱动 Controller.Tick(Mission, ms, dt)

TacticalMapController（总控）
  ├─ TerrainCache      烘焙地形/风险/单位 RGBA→GPU 纹理；WorldToUV 坐标换算
  ├─ FormationTracker  节流扫描编队 → FormationSnapshots（质心/朝向/队伍色）
  ├─ AgentTracker      扫描 Mission.Agents → 单位层 AgentRGBA + 密度
  ├─ CameraController   镜头目标/跟随
  ├─ OrderSystem        右键/Shift 集结点
  └─ TacticalMapLayer   Gauntlet 层：LoadMovie("TacticalMap")、接线 MinimapWidget、HitTest

MinimapWidget : ImageWidget
  └─ OnRender：按 1 的顺序把所有图层画出来
```
- `MinimapWidget` 是 `ImageWidget` 子类，被 GauntletUI **自动反射发现**（凡引用 `TaleWorlds.GauntletUI` 的程序集都会被扫描），**无需也不应手动 RegisterWidget**。
- 位置/尺寸由 XML 的 `HorizontalAlignment/MarginRight/MarginTop`（`MinimapPanel`）决定，不依赖 `Widget.PosOffset`（避免不同版本类型不一致 → `MissingMethodException`）。

### 2.3 构建与本地验证
1. 用游戏自带模块构建编译 `New_ZZZF`。csproj 的 TaleWorlds 引用指向 `$(GameFolder)\bin\Win64_Shipping_Client\`，**不要改成写死的 `Win64_Shipping_Client`，否则可能和构建工具覆盖的变量冲突**（见 `New_ZZZF.csproj` 14–15 行）。
2. 部署到 `Modules/New_ZZZF/`，确认 `GUI/Prefabs/TacticalMap.xml` 在模块 `GUI/Prefabs/` 下（电影名=`TacticalMap`，控件 Id=`MinimapPanel`/`MinimapTex`）。
3. 进战斗按 **N** 验收。应看到若干 `[TMap]` 游戏内消息：初始化 / 注入 / 正在绘制 / 切换显示。

### 2.4 跨版本兼容铁律（必读，否则必崩）
早期此功能 90% 的崩溃来自 **BL 版本间 `Rectangle2D` / `Widget` / `Texture` API 不一致**。现有代码已用以下手段规避，**二次开发务必沿用，禁止图省事直接写字段/属性名**：

1. **绘制区域取 `AreaRect.GetBoundingBox()`**（返回 `SimpleRectangle`，含 `X/Y/Width/Height`）。
   - 旧版本 `Widget.GlobalPosition` 返回 `Rectangle2D` 而非 `Vector2`，直接 `this.GlobalPosition` 会在 JIT 解析 `get_GlobalPosition(Vector2)` 时抛 `MissingMethodException`。`HitTestMinimap` 同理，已统一走 `GetBoundingBox()`。
2. **`Rectangle2D` 的位置/缩放/旋转写入全部走反射**（`MinimapWidget` 内 `ResolveRectMember` / `SetRectMember` / `SetRectPosition`）。
   - 编译引用集可能把 `LocalPosition/LocalScale/LocalRotation` 当**字段**（IL 走 `stfld`），而 1.4.6 运行时其实是**属性**，直接 `r.LocalScale = ...` → JIT 解析字段抛 `MissingFieldException`。
   - 策略：字段优先 → 否则同名属性；结果按名字缓存。**关键：`Rectangle2D` 是 struct，必须 box→设置→unbox 回写，否则改的是副本**。
3. **双 `System.Numerics.Vector2` 程序集问题**（`CoerceToTargetType`）。
   - 游戏自带一份 `Vector2`，本 mod 可能引用另一份；全名相同但程序集身份不同，反射 `SetValue` 会抛 `ArgumentException: Object of type 'System.Numerics.Vector2' cannot be converted to type 'System.Numerics.Vector2'`。
   - 解决：目标成员类型来自不同程序集时，用目标类型在其程序集内 `Activator.CreateInstance` 自行构造 `Vector2`。
4. **白色纹理：优先字节数组创建，反射查找为兜底**（`EnsureWhiteTexture`）。
   - 先 `TaleWorlds.Engine.Texture.CreateFromByteArray(new byte[]{255,255,255,255},1,1)` 在引擎侧造 1×1 纯白纹理（跨版本 100% 稳定，不依赖任何 sprite 名）；失败才回退反射查 `UIContext.SpriteData.GetSprite`（blank/white/…），再枚举所有 sprite 优先白名。**不要硬编码某个 sprite 名**。
5. **地形/风险/单位层用 GPU 烘焙纹理**（`TaleWorlds.Engine.Texture.CreateFromByteArray`）。
   - 先建底层 `Engine.Texture` 再包 `TwoDimension.Texture`（壳，无 Release）。真正释放靠 `Engine.Texture.Release()`，故单独持有 `_terrainETex/_riskETex/_agentETex` 以便复用/销毁时释放，避免 GPU 泄漏。
6. **`Widget.Width/Height` 读取用反射**（`WidgetSizeStr`，仅诊断用）。不同版本可能是属性/字段甚至命名 `Width/width`。

> 经验法则：**凡是 BL 引擎类型（Rectangle2D / Widget / Texture / Sprite）的字段或属性名，不要直接写死访问；要么走跨版本稳定的公开方法（如 `GetBoundingBox()`），要么走反射。** 直接写死是历次崩溃根因。

### 2.5 与早期版本相比已修好的坑（避免重复劳动）
- `Rectangle2D` 全部改走反射垫片（已无 `LocalPosition=` 直接写）。
- `DrawLine` 的朝向线改用 `SetRectMember(...,"LocalRotation",...)` 而非硬编码 position。
- 玩家标记改为**青环+黄心**、编队标记改为**队伍色方块+白边**，二者刻意区分（早期“难分辨”问题已解决）。
- `EnsureWhiteTexture` 改为**字节数组优先**（更稳），不再是纯反射。
- `OnRender` 已整体包 `try/catch`（首错仅提示一次 `[TMap] OnRender 异常: ...`），单帧异常不再拖垮整帧。
- 编队标记曾在某次重写中被误删，已在 `MinimapWidget.OnRender` 恢复；若日后“看不到编队标记”，先确认这段没再被删。

### 2.6 二次开发指引
- **加新图层**：`TacticalMapController` 备好数据数组 → `EnsureXxxTexture` 烘焙成纹理（参考 `EnsureTerrainTexture`）→ 在 `MinimapWidget.OnRender` 合适顺序调用绘制原语。
- **改标记样式**：玩家标记改“玩家（MainAgent）标记”段（青环 `DrawRectFrame` + 黄心 `DrawRect`）；编队标记改“编队标记”段（填充 `DrawRect` + 白边 `DrawRectFrame` + 朝向 `DrawLine`）。
- **绘制原语**：`DrawRect`（实心矩形）、`DrawLine`（带角度条）、`DrawDiamond`（四线菱形）、`DrawRectFrame`（四边描边）、`DrawTexture`（整张纹理拉伸）。
- **改尺寸/位置**：调 `TacticalSettings.MapSize`（代码）+ `TacticalMap.xml` 里 `MinimapPanel` 的 `MarginRight/MarginTop`（保持一致）。
- **加子功能开关**：`TacticalFeature` 枚举 + `FeatureGate.IsEnabled` 加分支 + `TacticalSettings` 加字段。
- ⚠ **任何 `Rectangle2D` 的位置/缩放/旋转都必须经 `SetRectPosition` / `SetRectMember`，禁止直接 `r.LocalPosition = ...`。**

### 2.7 已知问题 & 诊断速查
- **`tmap_diag.log` 噪声**：`MinimapWidget` 把 `[TMap]` 诊断同时写游戏内消息和 `我的文档/tmap_diag.log`（前几帧/前十几秒较密，有帧计数节流）。要静默就把对应 `Diag(...)` 注释掉。
- **图例/快捷键文案**：见 §1 复查点，纯文案。
- **性能**：地形/风险仅初始化或缓存切换时烤一次；单位层仅 `AgentDataVersion` 变化时重建（≈6.6Hz）；密度热力每帧扫 48×48 网格，开销极小。

| 现象 | 可能原因 | 排查 |
|------|----------|------|
| 按 N 没反应 / 无 `[TMap]` 消息 | `EnableMinimap=false` 或未注入 MissionBehavior | 查 `SubModule.OnMissionBehaviorInitialize` 是否调 `TacticalMapBootstrap.OnMissionStart`；确认 `FeatureGate.Enabled` |
| 小地图出现但空白/黑 | `TerrainCache` 未烘焙或纹理创建失败 | 看 `[TMap] 初始化` 消息的 `baked/err`；查 `EnsureTerrainTexture` 的 `CreateFromByteArray` |
| 标记全是方块且重叠难辨 | 编队/玩家标记样式问题 | 见 §1、§2.6；查 `DrawRectFrame`/`DrawDiamond` |
| `MissingFieldException: Rectangle2D.LocalPosition` | 又直接写了 `r.LocalPosition=` | 全局搜 `LocalPosition`/`localPosition`/`AreaRect` 直接访问，改回反射 `SetRectPosition` |
| `MissingMethodException: get_GlobalPosition(Vector2)` | 又直接写 `this.GlobalPosition` | 改用 `AreaRect.GetBoundingBox()` |
| 编译 `CS1061: Rectangle2D 未包含 localPosition` | 编译引用集较新（大写/属性），代码却写小写字段 | 同 §2.4 第 2 点，走反射 |
| 反射 `ArgumentException: Vector2 不能转换为 Vector2` | 双 Vector2 程序集 | 已在 `CoerceToTargetType` 处理，确认没绕过它 |
| `[TMap] OnRender 异常: ...` | OnRender 内某 API 在该版不兼容 | 看异常信息，基本都是 Rectangle2D/Texture API 问题，回到 §2.4 的反射方案 |
