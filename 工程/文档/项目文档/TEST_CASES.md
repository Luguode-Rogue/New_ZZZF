# 测试用例 - ProjectileTrajectorySystem

> **概要**：本文档提供ProjectileTrajectorySystem Mod的完整游戏内手动测试用例，覆盖全部8大功能模块（玩家弹道、攻城武器弹道、敌人弹道追踪、飞行投射物轨迹、敌人高亮描边、抬头虚化、移动目标预瞄、近敌慢动作）+双渲染管线切换+设置系统+海战DLC支持+集成测试。每个测试用例包含前置条件、操作步骤、预期结果、清理步骤。适合QA测试或开发者自测使用。
> 
> Bannerlord Mod 运行在游戏引擎内部，无法使用传统单元测试框架。以下为**游戏内手动测试用例**，覆盖全部 8 大功能模块。

## 1. 玩家弹道预测

**对应代码**: `Systems/PlayerTrajectorySystem.UpdateRangeWeapon()`

### TC-1.1 弓箭弹道显示

| 项目 | 内容 |
|------|------|
| 前置 | 进入战场，装备弓和箭矢 |
| 步骤 | 1. 装备弓作为主武器<br>2. 按住鼠标右键进入瞄准<br>3. 按住鼠标左键拉弓 |
| 预期 | 屏幕上显示从玩家视角出发的预测弹道线（DebugLine 彩虹色线段 或 GameEntity 图标点） |
| 清理 | 松开鼠标，弹道消失 |

### TC-1.2 弩弹道显示

| 项目 | 内容 |
|------|------|
| 前置 | 装备弩和弩箭 |
| 步骤 | 1. 装备弩<br>2. 按住右键瞄准 |
| 预期 | 显示弩的预测弹道（弩已上弦，无需拉弓即显示） |

### TC-1.3 投掷武器弹道显示

| 项目 | 内容 |
|------|------|
| 前置 | 装备投掷标枪/飞斧 |
| 步骤 | 1. 装备投掷武器<br>2. 按住右键瞄准 |
| 预期 | 显示较短的投掷弹道（投掷武器初速低、阻力大） |

### TC-1.4 "仅准备攻击时显示" 开关

| 项目 | 内容 |
|------|------|
| 前置 | MCM 设置中 `EnableAttackReadyRestriction = true` |
| 步骤 | 1. 装备弓<br>2. 仅按右键瞄准（不拉弓）<br>3. 再按住左键拉弓 |
| 预期 | 步骤2不显示弹道；步骤3显示弹道 |
| 变体 | 设置 `EnableAttackReadyRestriction = false` 后，步骤2也应显示弹道 |

### TC-1.5 非远程武器不显示弹道

| 项目 | 内容 |
|------|------|
| 前置 | 装备近战武器 |
| 步骤 | 挥舞近战武器 |
| 预期 | 不显示任何弹道线 |

### TC-6.2 攻城武器仰射虚化

| 项目 | 内容 |
|------|------|
| 前置 | `EnableLookUpBlur = true`，操作投石车 |
| 步骤 | 调高投石车仰角 |

### TC-6.3 平滑过渡效果

| 项目 | 内容 |
|------|------|
| 前置 | TC-6.1 已触发，角色处于虚化状态 |
| 步骤 | 将视角恢复水平 |
| 预期 | alpha 不是瞬间恢复，而是以 ~0.15 的平滑因子渐变回 1.0 |

### TC-6.4 开关切换

| 项目 | 内容 |
|------|------|

| 步骤 | MCM 中关闭 `EnableLookUpBlur` |

## 10. 设置系统

## 12. 集成测试

### TC-12.1 全功能同时启用

| 项目 | 内容 |
|------|------|
| 前置 | 所有 MCM 设置均为 true，`UseDebugLineDisplay = true` |
| 步骤 | 进入大型战场，装备弓，有多个敌人同时瞄准玩家 |
| 预期 | 同时显示：玩家弹道、敌人红色弹道、投射物绿色轨迹、敌人红色描边、仰射虚化、预瞄标记、近敌慢动作 |

### TC-12.2 战斗结束后清理

| 项目 | 内容 |
|------|------|
| 前置 | 战斗进行中，弹道/描边/虚化均生效 |
| 步骤 | 战斗结束，退出战场 |
| 预期 | 所有 GameEntity 归还池、描边清除、alpha 恢复（`OnEndMission` 触发 `GameEntityPool.Clear()` + `EnemyOutlineSystem.Clear()` + `AlphaBlurSystem.RestoreAll()`） |

### TC-12.3 多场战斗连续进行

| 项目 | 内容 |
|------|------|
| 前置 | 完成第一场战斗 |
| 步骤 | 不重启游戏，直接进入第二场战斗 |
| 预期 | 所有功能正常，无残留弹道/描边/虚化状态 |

### TC-12.4 性能压力测试

| 项目 | 内容 |
|------|------|
| 前置 | 大型攻城战 (200+ agent)，DebugLine 模式 |
| 步骤 | 同时有 10 个敌人瞄准玩家，玩家操作投石车 |
| 预期 | 帧率下降在可接受范围内（对象池 + 颜色缓存优化起效） |

### TC-12.5 API 兼容性验证

| 项目 | 内容 |
|------|------|
| 前置 | 有其他 Mod 依赖本 Mod 的 public API |
| 步骤 | 验证以下方法仍可正常外部调用：<br>- `ProjectileTrajectorySystem.UpdateTrajectory(agent, siege)`<br>- `ProjectileTrajectorySystem.GetWeaponAirFriction(agent, out _)`<br>- `ProjectileTrajectorySystem.CalculateLeadPosition(player, target, speed)`<br>- `SkillSystemBehavior.SetSiegeTargetAlpha(siege, alpha)`<br>- `SkillSystemBehavior.WoW_Line` 字典访问 |
| 预期 | 所有 public API 签名和功能与重构前完全一致 |
