# 战场HUD进度条 - 实现文档

**日期**: 2026-06-15  
**任务**: 将战场HUD从纯文本显示升级为FillBarWidget进度条样式  
**模组**: New_ZZZF  

---

## 一、功能目标

**原状态**: 战场HUD使用纯文本显示法力/耐力数值，用户体验差。

**目标效果**: 改为类原版血条的FillBarWidget进度条，类似WoW模组的实现。

**最终效果**:
- 右下角显示两个进度条（法力条蓝色、耐力条橙色）
- 进度条上方显示文字 "法力 50/100" "耐力 80/100"
- 实时更新，跟随玩家状态变化

## 二、实现步骤（完整流程）

### 步骤1：创建 ViewModel（数据绑定层）

**文件**: `工程/New_ZZZF/GUI/NewZZZF_MissionAgentStatusVM.cs`

**作用**: 定义UI需要显示的数据属性，并通过 `OnPropertyChanged` 通知UI更新。

**关键代码**:

```csharp
public class NewZZZF_MissionAgentStatusVM : ViewModel
{
    // 法力当前值（用于进度条填充比例）
    private float _manaCurrent = 0f;
    
    // 法力最大值（用于计算100%）
    private float _manaMax = 100f;
    
    // 法力文字（显示在进度条上方）
    private string _manaText = "法力  0/100";

    // 耐力当前值
    private float _staminaCurrent = 0f;
    
    // 耐力最大值
    private float _staminaMax = 100f;
    
    // 耐力文字
    private string _staminaText = "耐力  0/100";

    // 数据绑定属性（必须加 [DataSourceProperty] 特性）
    [DataSourceProperty]
    public float ManaCurrent
    {
        get => _manaCurrent;
        set
        {
            if (_manaCurrent != value)
            {
                _manaCurrent = value;
                OnPropertyChangedWithValue(value, nameof(ManaCurrent));  // 通知UI更新
            }
        }
    }

    [DataSourceProperty]
    public float ManaMax { get => _manaMax; set { ... } }

    [DataSourceProperty]
    public string ManaText { get => _manaText; set { ... } }

    [DataSourceProperty]
    public float StaminaCurrent { get => _staminaCurrent; set { ... } }

    [DataSourceProperty]
    public float StaminaMax { get => _staminaMax; set { ... } }

    [DataSourceProperty]
    public string StaminaText { get => _staminaText; set { ... } }

    // 从 AgentSkillComponent 更新所有绑定值
    public void UpdateFromComponent(AgentSkillComponent comp)
    {
        if (comp == null)
        {
            ManaCurrent = 0;
            StaminaCurrent = 0;
            ManaText = "法力  --/--";
            StaminaText = "耐力  --/--";
            return;
        }

        ManaCurrent = comp._currentMana;
        StaminaCurrent = comp._currentStamina;
        ManaText = $"法力  {(int)comp._currentMana}/{(int)ManaMax}";
        StaminaText = $"耐力  {(int)comp._currentStamina}/{(int)StaminaMax}";
    }
}
```

**关键点**:
1. 所有需要绑定的属性必须加 `[DataSourceProperty]` 特性
2. 属性setter中必须调用 `OnPropertyChangedWithValue` 通知UI更新
3. `UpdateFromComponent` 方法从 `AgentSkillComponent` 读取数据并更新属性

### 步骤2：创建 XML 布局（UI层）

**文件**: `GUI/Prefabs/NewZZZF_MissionAgentStatus.xml`

**作用**: 定义UI的布局、样式、数据绑定关系。

**完整代码**:

```xml
<Prefab>
  <Window>
    <!-- 根Widget必须有Name属性，值与LoadMovie的第一个参数匹配 -->
    <Widget Name="NewZZZF_MissionAgentStatus"
            WidthSizePolicy="Fixed" HeightSizePolicy="Fixed"
            SuggestedWidth="220" SuggestedHeight="54"
            HorizontalAlignment="Right" VerticalAlignment="Bottom"
            MarginRight="30" MarginBottom="280">
      <Children>

        <!-- ===== 法力条 ===== -->
        <Widget WidthSizePolicy="Fixed" HeightSizePolicy="Fixed"
                SuggestedWidth="220" SuggestedHeight="22"
                HorizontalAlignment="Left" VerticalAlignment="Top"
                PositionYOffset="0"
                Sprite="BlankWhiteSquare_9" Color="#151515FF">
          <Children>
            <!-- 法力填充进度 -->
            <FillBarWidget WidthSizePolicy="StretchToParent" HeightSizePolicy="StretchToParent"
                           FillWidget="ManaFill" ClipContents="true"
                           InitialAmountAsFloat="@ManaCurrent" MaxAmountAsFloat="@ManaMax">
              <Children>
                <Widget Id="ManaFill" WidthSizePolicy="StretchToParent" HeightSizePolicy="StretchToParent"
                        HorizontalAlignment="Left"
                        Sprite="BlankWhiteSquare_9" Color="#3366CCFF" />
              </Children>
            </FillBarWidget>
            <!-- 法力文字居中 -->
            <TextWidget WidthSizePolicy="StretchToParent" HeightSizePolicy="StretchToParent"
                        Text="@ManaText"
                        HorizontalAlignment="Center" VerticalAlignment="Center"
                        Brush="MPLobbyCurrentCharacterDifficultyTitle" />
          </Children>
        </Widget>

        <!-- ===== 耐力条 ===== -->
        <Widget WidthSizePolicy="Fixed" HeightSizePolicy="Fixed"
                SuggestedWidth="220" SuggestedHeight="22"
                HorizontalAlignment="Left" VerticalAlignment="Top"
                PositionYOffset="28"
                Sprite="BlankWhiteSquare_9" Color="#151515FF">
          <Children>
            <!-- 耐力填充进度 -->
            <FillBarWidget WidthSizePolicy="StretchToParent" HeightSizePolicy="StretchToParent"
                           FillWidget="StaminaFill" ClipContents="true"
                           InitialAmountAsFloat="@StaminaCurrent" MaxAmountAsFloat="@StaminaMax">
              <Children>
                <Widget Id="StaminaFill" WidthSizePolicy="StretchToParent" HeightSizePolicy="StretchToParent"
                        HorizontalAlignment="Left"
                        Sprite="BlankWhiteSquare_9" Color="#CC9933FF" />
              </Children>
            </FillBarWidget>
            <!-- 耐力文字居中 -->
            <TextWidget WidthSizePolicy="StretchToParent" HeightSizePolicy="StretchToParent"
                        Text="@StaminaText"
                        HorizontalAlignment="Center" VerticalAlignment="Center"
                        Brush="MPLobbyCurrentCharacterDifficultyTitle" />
          </Children>
        </Widget>

      </Children>
    </Widget>
  </Window>
</Prefab>
```

**关键点**:

1. **根Widget必须有Name属性**: `Name="NewZZZF_MissionAgentStatus"`，这个值必须与 `LoadMovie` 的第一个参数完全一致。

2. **FillBarWidget工作原理**:
   - `InitialAmountAsFloat="@ManaCurrent"`: 绑定当前值
   - `MaxAmountAsFloat="@ManaMax"`: 绑定最大值
   - `FillWidget="ManaFill"`: 指定填充子Widget的Id
   - `ClipContents="true"`: 启用裁剪，根据比例裁剪子Widget
   - 计算方式: `填充比例 = InitialAmountAsFloat / MaxAmountAsFloat`
   - 例如: ManaCurrent=50, ManaMax=100 → 填充50%宽度

3. **填充子Widget必须用StretchToParent**:
   - 错误: `WidthSizePolicy="Fixed" SuggestedWidth="220"`
   - 正确: `WidthSizePolicy="StretchToParent"`
   - 原因: FillBarWidget通过裁剪子Widget显示进度，子Widget必须撑满父容器，裁剪才有效

4. **布局位置**:
   - `HorizontalAlignment="Right" VerticalAlignment="Bottom"`: 右下角
   - `MarginRight="30" MarginBottom="280"`: 距离右边缘30px，距离底边缘280px（避免与原版血条重叠）

5. **颜色值格式**: `#AARRGGBB`（Alpha, Red, Green, Blue）
   - 法力条: `#3366CCFF` = 不透明蓝色
   - 耐力条: `#CC9933FF` = 不透明橙色
   - 背景: `#151515FF` = 不透明深灰色

### 步骤3：创建 View（逻辑层）

**文件**: `工程/New_ZZZF/GUI/NewZZZF_MissionAgentStatusView.cs`

**作用**: 管理UI的生命周期（创建、更新、销毁），连接ViewModel和XML。

**完整代码**:

```csharp
using System;
using System.IO;
using TaleWorlds.Engine.GauntletUI;
using TaleWorlds.Library;
using TaleWorlds.MountAndBlade;
using TaleWorlds.MountAndBlade.View;
using TaleWorlds.ScreenSystem;

namespace New_ZZZF
{
    /// <summary>
    /// 战场法力/耐力 HUD 视图
    /// 继承 MissionView，在 OnMissionScreenTick 中懒初始化 UI
    /// </summary>
    public class NewZZZF_MissionAgentStatusView : MissionView
    {
        private GauntletLayer _gauntletLayer;
        private NewZZZF_MissionAgentStatusVM _dataSource;
        private GauntletMovieIdentifier _movie;
        private bool _initialized;
        private bool _triedInit;
        private int _tickCount;
        private readonly string _logPath = "../../Modules/New_ZZZF/Logs/NewZZZF_HUD.txt";

        private void Log(string msg)
        {
            try { File.AppendAllText(_logPath, $"[{DateTime.Now:HH:mm:ss.fff}] {msg}{Environment.NewLine}"); } catch { }
        }

        // MissionView 的每帧回调——MissionScreen 激活时每帧调用
        public override void OnMissionScreenTick(float dt)
        {
            _tickCount++;
            var missionScreen = this.MissionScreen;

            if (_tickCount <= 5 || _tickCount % 300 == 0)
            {
                Log($"OnMissionScreenTick #{_tickCount}, Agent.Main={Agent.Main != null}, MissionScreen={missionScreen != null}");
            }

            // 懒初始化：Agent.Main 存在且 MissionScreen 存在
            if (!_triedInit && Agent.Main != null && missionScreen != null)
            {
                _triedInit = true;
                Log($"===== 懒初始化触发（Agent.Main={Agent.Main.Name}）=====");
                TryInitializeUI(missionScreen);
            }

            // 每帧更新 VM 数据
            if (_dataSource != null && Agent.Main != null)
            {
                SkillSystemBehavior.ActiveComponents.TryGetValue(Agent.Main.Index, out var comp);
                _dataSource.UpdateFromComponent(comp);
            }

            base.OnMissionScreenTick(dt);
        }

        private void TryInitializeUI(MissionScreen missionScreen)
        {
            if (_initialized)
            {
                Log("TryInitializeUI: 已初始化，跳过");
                return;
            }
            _initialized = true;

            Log("===== 开始初始化 HUD UI =====");
            Log($"MissionScreen: {missionScreen != null}");
            Log($"Agent.Main: {Agent.Main != null}");

            try
            {
                // 1. 创建 ViewModel
                _dataSource = new NewZZZF_MissionAgentStatusVM();
                Log($"VM 创建: {_dataSource != null}");

                // 2. 创建 GauntletLayer（UI层）
                _gauntletLayer = new GauntletLayer("NewZZZF_MissionHUD", 30, false)
                {
                    IsFocusLayer = false  // 不拦截输入
                };
                Log($"GauntletLayer 创建: {_gauntletLayer != null}");

                // 3. 加载 XML 布局（LoadMovie 的第一个参数 = XML根Widget的Name属性）
                _movie = _gauntletLayer.LoadMovie("NewZZZF_MissionAgentStatus", _dataSource);
                Log($"LoadMovie 结果: {(_movie == null ? "NULL!!!" : "成功")}");

                if (_movie == null)
                {
                    InformationManager.DisplayMessage(new InformationMessage("[New_ZZZF] LoadMovie 返回 null，请检查 GUI/Prefabs/NewZZZF_MissionAgentStatus.xml"));
                    Log("LoadMovie 失败，中止初始化");
                    _gauntletLayer = null;
                    _dataSource = null;
                    _initialized = false;
                    return;
                }

                // 4. 将 Layer 添加到 MissionScreen（显示UI）
                missionScreen.AddLayer(_gauntletLayer);
                Log("AddLayer 成功");

                InformationManager.DisplayMessage(new InformationMessage("[New_ZZZF] HUD 已加载，查看右下角"));
                Log("===== HUD UI 初始化完成 =====");
            }
            catch (Exception ex)
            {
                Log($"初始化异常: {ex}");
                InformationManager.DisplayMessage(new InformationMessage($"[New_ZZZF] 初始化异常: {ex.Message}"));
                _initialized = false;
                _triedInit = false;
            }
        }

        public override void OnRemoveBehavior()
        {
            Log($"OnRemoveBehavior 触发 (tick总数={_tickCount})，开始清理");
            CleanupLayer();
            _dataSource = null;
            base.OnRemoveBehavior();
            Log("OnRemoveBehavior 完成");
        }

        private void CleanupLayer()
        {
            var missionScreen = this.MissionScreen;

            if (_gauntletLayer != null && _movie != null)
            {
                _gauntletLayer.ReleaseMovie(_movie);
                _movie = null;
                Log("ReleaseMovie 完成");
            }

            if (_gauntletLayer != null)
            {
                if (missionScreen != null)
                {
                    missionScreen.RemoveLayer(_gauntletLayer);
                }
                _gauntletLayer = null;
                Log("RemoveLayer 完成");
            }
        }
    }
}
```

**关键点**:

1. **继承 MissionView**:
   - 正确基类是 `MissionView`（不是 `MissionBehavior`）
   - `MissionView` 自带 `MissionScreen` 属性，无需手动获取
   - 每帧回调是 `OnMissionScreenTick`（不是 `OnMissionTick`）

2. **懒初始化模式**:
   - 不在生命周期回调中初始化（时机不可靠）
   - 在 `OnMissionScreenTick` 中检查条件：`Agent.Main != null && this.MissionScreen != null`
   - 条件满足时才初始化，避免空引用

3. **获取 MissionScreen**:
   - `MissionView` 自带 `this.MissionScreen` 属性，直接用

4. **GauntletLayer 创建**:
   - 第一个参数: Layer名称（任意字符串，用于标识）
   - 第二个参数: 渲染顺序（数字越大越靠前）
   - 第三个参数: 是否使用局部排序（一般false）

5. **LoadMovie 参数**:
   - 第一个参数: XML根Widget的Name属性值（必须完全匹配）
   - 第二个参数: ViewModel实例（数据源）

6. **每帧更新数据**:
   - 在 `OnMissionScreenTick` 中调用 `_dataSource.UpdateFromComponent(comp)`
   - 从 `SkillSystemBehavior.ActiveComponents` 字典中根据 `Agent.Main.Index` 获取组件

7. **清理资源**:
   - 在 `OnRemoveBehavior` 中释放Movie和Layer
   - 必须先 `ReleaseMovie`，再 `RemoveLayer`
   - `this.MissionScreen` 在战斗结束后可能为null，需要判断

### 步骤4：注册 View（入口）

**文件**: `工程/New_ZZZF/SubModule.cs`

**作用**: 在战场加载时注册自定义的 MissionBehavior。

**关键代码**:

```csharp
public override void OnMissionBehaviorInitialize(Mission mission)
{
    base.OnMissionBehaviorInitialize(mission);

    // 添加自定义的 MissionBehavior 到当前任务
    mission.AddMissionBehavior(new SkillSystemBehavior());
    mission.AddMissionBehavior(new MountedSlashCameraMissionLogic());
    mission.AddMissionBehavior(new HeroChangeMissionBehavior());
    mission.AddMissionBehavior(new AffixMissionBehavior());
    mission.AddMissionBehavior(new NewZZZF_MissionAgentStatusView());  // 注册HUD视图
    
    InformationManager.DisplayMessage(new InformationMessage(
        "[New_ZZZF] 技能系统已激活！"));
}
```

**关键点**:
1. `OnMissionBehaviorInitialize` 是 SubModule 中注册 MissionBehavior 的入口
2. 用 `mission.AddMissionBehavior()` 注册（不是 `AddMissionView`）
3. 注册顺序不重要，每个 Behavior 独立工作

## 三、数据流图

```
玩家Agent状态变化
    ↓
AgentSkillComponent._currentMana / _currentStamina 更新
    ↓
OnMissionTick 每帧调用
    ↓
_dataSource.UpdateFromComponent(comp)  // 从组件读取数据
    ↓
ManaCurrent = comp._currentMana      // ViewModel属性更新
    ↓
OnPropertyChangedWithValue()          // 通知UI绑定属性变化
    ↓
XML中的 @ManaCurrent 绑定自动更新
    ↓
FillBarWidget 重新计算填充比例
    ↓
UI进度条显示更新
```

## 五、常见问题排查

### 问题1：UI完全不显示

**排查步骤**:
1. 检查日志文件是否有 `OnMissionTick` 输出（确认回调被调用）
2. 检查日志是否有 `懒初始化触发`（确认初始化条件满足）
3. 检查日志是否有 `LoadMovie 结果: 成功`（确认XML加载成功）
4. 检查日志是否有 `AddLayer 成功`（确认UI已添加）
5. 如果以上都有，检查XML布局位置是否正确（可能被遮挡或超出屏幕）

### 问题2：LoadMovie 返回 null

**可能原因**:
1. XML文件不存在于 `GUI/Prefabs/` 目录
2. XML根Widget没有Name属性
3. Name属性值与LoadMovie第一个参数不匹配

**解决方案**:
- 确认XML文件路径正确
- 确认根Widget有 `Name="NewZZZF_MissionAgentStatus"`
- 确认LoadMovie第一个参数是 `"NewZZZF_MissionAgentStatus"`

### 问题3：进度条不更新

**可能原因**:
1. ViewModel属性没有加 `[DataSourceProperty]` 特性
2. 属性setter中没有调用 `OnPropertyChangedWithValue`
3. `OnMissionTick` 中没有调用 `UpdateFromComponent`
4. `Agent.Main` 为null，导致无法获取组件

**解决方案**:
- 检查ViewModel所有属性都有 `[DataSourceProperty]`
- 检查属性setter中调用了 `OnPropertyChangedWithValue`
- 检查 `OnMissionTick` 中有更新逻辑
- 检查日志确认 `Agent.Main` 不为null

## 六、关键技术点总结

| 技术点 | 说明 | 坑点 |
|--------|------|------|
| 基类选择 | 正确基类是 `MissionView` | 误用 `MissionBehavior` 会缺少 `MissionScreen` 属性，且 `BehaviorType` 是抽象属性必须实现 |
| 每帧回调 | `MissionView` 用 `OnMissionScreenTick` | 误用 `OnMissionTick`（那是 `MissionBehavior` 的回调） |
| 获取MissionScreen | `this.MissionScreen`（MissionView自带） | MissionBehavior 中才需要用 `ScreenManager.TopScreen as MissionScreen` |
| XML根Widget的Name属性 | 必须与LoadMovie第一个参数完全匹配 | 缺少Name属性会导致LoadMovie失败 |
| FillBarWidget子Widget宽度 | 必须用StretchToParent | 固定宽度会导致裁剪失效 |
| ViewModel属性绑定 | 必须加[DataSourceProperty]并调用OnPropertyChanged | 忘记加特性会导致UI不更新 |
| 懒初始化模式 | 在OnMissionScreenTick中检查条件后初始化 | 生命周期回调时机不可靠 |
