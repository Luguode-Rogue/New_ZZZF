# 技能界面闪退 Bug 修复 — 大地图暂停机制

**日期**：2026-06-09  
**严重级别**：高（AccessViolationException 必崩溃）  
**状态**：✅ 已修复，验证通过

---

## 1. 问题概述

打开技能界面（CustomSkillScreen）后，游戏在几秒内闪退，抛出 `System.AccessViolationException`。

### 崩溃堆栈

```
System.AccessViolationException
  HResult=0x80004003
  Message=Attempted to read or write protected memory.

[托管到本机的转换]
NavalDLC.View.dll!NavalMobilePartyVisual.TickOars(float dt, float realDt) 行 909
NavalDLC.View.dll!NavalMobilePartyVisual.UpdateEntityPosition(float dt, float realDt, bool isVisible) 行 405
NavalDLC.View.dll!NavalMobilePartyVisual.Tick(float dt, float realDt, ...) 行 327
NavalDLC.View.dll!NavalMobilePartyVisualManager.OnTick.AnonymousMethod__0(...) 行 59
```

---

## 2. 根因分析

### 直接原因

`TickOars` 方法访问了已释放的本机内存（船桨可视化相关的引擎对象）。

### 深层原因

CustomSkillScreen 打开时，大地图时间仍在推进：

```
CustomSkillScreen 打开
  → 只 PushScreen，未 Push GameState
  → MapState 仍是 ActiveState，OnTick() 持续运行
  → TickMapTime() 推进世界时间
  → 队伍移动、创建/销毁实体
  → NavalMobilePartyVisual.Tick() → TickOars() 访问已销毁的本机对象
  → AccessViolationException 崩溃
```

### 对比原生界面

物品栏等原生界面打开时：
```
InventoryState.Push
  → MapState 自动 Deactivate
  → MapState.OnTick() 停止
  → 世界时间冻结
  → 不会崩溃
```

---

## 3. 解决方案

采用骑砍2原生界面的标准模式：**`RegisterActiveStateDisableRequest` / `UnregisterActiveStateDisableRequest`**。

这个机制不需要创建新的 GameState，直接将当前 ActiveState（MapState）的 `OnTick` 降级为 `OnIdleTick`，冻结时间推进。

### 参考代码（已有先例）

| 界面 | 文件 | 用法 |
|------|------|------|
| GauntletBodyGeneratorScreen | TaleWorlds.MountAndBlade.GauntletUI | OnInitialize 注册 / OnFinalize 取消 |
| GauntletBarberScreen | 同上 | 同上 |
| GauntletSaveLoadScreen | 同上 | 同上 |
| OptionsVM | TaleWorlds.MountAndBlade.ViewModelCollection | 构造注册 / CloseScreen 取消 |
| MissionGauntletEscapeMenuBase | TaleWorlds.MountAndBlade.GauntletUI | OnActivate 注册 / OnDeactivate 取消 |
| MapScreen (Esc菜单) | SandBox.View | 打开注册 / 关闭取消 |

### 实际修改

**文件**：`E:\SteamLibrary\steamapps\common\Mount & Blade II Bannerlord\Modules\New_ZZZF\工程\New_ZZZF\GUI\CustomSkillScreen.cs`

**改动 1**：添加命名空间
```csharp
using TaleWorlds.Core;  // Game 类所在命名空间
```

**改动 2**：OnInitialize() 末尾 — 注册暂停请求
```csharp
// 暂停大地图时间推进（参考 BarberScreen/SaveLoadScreen 的标准做法）
Game.Current.GameStateManager.RegisterActiveStateDisableRequest(this);
```

**改动 3**：OnFinalize() 开头 — 取消暂停请求
```csharp
// 恢复大地图时间推进
Game.Current.GameStateManager.UnregisterActiveStateDisableRequest(this);
```

---

## 4. 机制说明

```
GameStateManager._activeStateDisableRequests (List<WeakReference>)
                                            │
                    ┌───────────────────────┼───────────────────────┐
                    ▼                                               ▼
            列表非空时                                        列表为空时
        ActiveState 只跑 OnIdleTick                       ActiveState 正常跑 OnTick
        （时间冻结，实体不创建/销毁）                        （时间正常推进）
```

- `RegisterActiveStateDisableRequest` — 向列表添加 WeakReference(this)
- `UnregisterActiveStateDisableRequest` — 从列表移除
- 使用 `WeakReference` 保证请求者被 GC 回收后不会残留

---

## 5. 验证结果

修改后重新编译，打开技能界面停留数分钟，**不再复现崩溃**。大地图时间正确冻结，关闭界面后时间恢复正常推进。

## 6. 排查过程中的误区

| 误区 | 说明 |
|------|------|
| 缺少 `using TaleWorlds.Core;` | `Game` 类在该命名空间，初次编译报 "当前上下文中不存在名称 Game" |
| 另一可用路径 `GameStateManager.Current` | 也有效（GauntletSceneNotification 使用此路径），但 `Game.Current.GameStateManager` 更标准 |
