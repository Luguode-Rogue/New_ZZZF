## Bug：存档读档后词缀丢失（已修复）

### 根本原因
`AffixCampaignBehavior.SyncData()` 是**空实现**，只打印了日志，没有调用 `dataStore.SyncData<T>()` 来序列化/反序列化 `ItemAffixMap` 和 `IsInitialized`。

### 第一次尝试（失败）
直接 `dataStore.SyncData<Dictionary<string, AffixInstance>>` → **游戏保存崩溃**（无法创建存档）。

**失败原因**：Bannerlord 的 SaveSystem 对于自定义 class 类型需要 `SaveableTypeDefiner` + 自动生成的序列化代码（构建工具生成），Mod 无法使用此机制。`Dictionary<K,V>` 虽是支持的容器，但其中的值类型 `AffixInstance` 未注册 → 运行时崩溃。

### 第二次尝试（成功）：List&lt;string&gt; 文本序列化

**策略**：绕开自定义类型的直接序列化，用 `List<string>`（Bannerlord 原生支持）保存文本表示，读档时解析还原。

**代码结构**：

```csharp
// 存档字段（仅用 Bannerlord 原生支持的类型）
[SaveableField(1)] private List<string> _serializedAffixData = new List<string>();
[SaveableField(2)] private bool _serializedIsInitialized;

// 运行时字典（不直接存档）
public Dictionary<string, AffixInstance> ItemAffixMap = new Dictionary<string, AffixInstance>();
public bool IsInitialized;

// SyncData
public override void SyncData(IDataStore dataStore)
{
    if (dataStore.IsSaving)
    {
        _serializedAffixData = SerializeAffixMap();  // 字典 → 字符串列表
        _serializedIsInitialized = IsInitialized;
    }
    dataStore.SyncData<List<string>>("_serializedAffixData", ref _serializedAffixData);
    dataStore.SyncData<bool>("_serializedIsInitialized", ref _serializedIsInitialized);
    if (dataStore.IsLoading)
    {
        IsInitialized = _serializedIsInitialized;
        ItemAffixMap = DeserializeAffixMap(_serializedAffixData);  // 字符串列表 → 字典
        // 重建 [NonSerialized] 缓存
    }
}
```

**每行格式**：
```
InstanceId|BaseItemId|ItemLevel|Rarity|prefixId1,prefixId2|suffixId1,suffixId2|statKey1=statVal1;statKey2=statVal2
```

**注意**：读档后需用 `ItemAffixMap.Keys.ToList()` 遍历并回写（因为 struct 值拷贝问题）。

## 踩坑记录：SaveableProperty vs SaveableField

- `[SaveableProperty]` 只对**属性**有效 → 字段上用会报 CS0592
- `[SaveableField]` 对**字段**有效
- 两种都只是标记，要真正序列化必须在 `SyncData` 中调用 `dataStore.SyncData<T>()`
