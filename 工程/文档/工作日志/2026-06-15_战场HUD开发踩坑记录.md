**原因**: 项目中使用的API版本已过时，MBCommon.GetTime在新版本中不存在

**错误代码**:

```csharp
float currentTime = Mission.Current.CurrentTime;
```

**经验教训**: 
- 不要假设API存在，先搜索项目中的用法
- 骑砍2的GameTime相关API统一用Mission.Current.CurrentTime

### 返工2：变量名作用域冲突

**现象**: 编译错误 CS0136 - 不能在此范围内声明名为'enemies'的局部变量，因为该名称已在封闭范围中用于定义局部变量或参数


**经验教训**: 
- C#不允许在嵌套作用域中声明同名变量
- 注意变量作用域，避免重复命名

### 返工3：MissionView注册方式错误导致NullReferenceException

**现象**: 运行时抛出NullReferenceException，GauntletLayer初始化失败

**原因**: 
- 在SubModule中用`AddMissionBehavior()`注册`MissionView`子类
- MissionView的生命周期方法（如OnMissionScreenInitialize）不会被正确调用
- MissionView和MissionBehavior是两种不同的注册体系

**错误代码**:
```csharp
// SubModule.cs
public override void OnMissionBehaviorInitialize(Mission mission)
{
    mission.AddMissionBehavior(new NewZZZF_MissionAgentStatusView());  // 错误！
}
```

**正确写法**:
```csharp
// 方案1：如果View继承MissionBehavior，可以用AddMissionBehavior
public class NewZZZF_MissionAgentStatusView : MissionBehavior  // 改继承
{
    // 然后使用AddMissionBehavior注册
}

// 方案2：如果View继承MissionView，应该用AddMissionView
// 但MissionView的注册方式更复杂，不推荐
```

**经验教训**: 
- MissionView和MissionBehavior不能混用注册方式
- MissionBehavior更可靠，推荐使用
- 如果一定要用MissionView，需要研究正确的注册方式

### 返工5：UI不显示（生命周期调用时机问题）

**现象**: 编译通过，游戏内没有报错，但UI完全不显示

**原因**: 
- 最初在OnMissionBehaviorInitialize中初始化UI
- 但此时Agent.Main可能还未创建（Behavior注册早于Agent创建）
- 后来改用OnAgentCreated回调，但又可能Agent已经创建完毕才注册Behavior
- 总之：生命周期调用时机不可靠

**错误代码**:
```csharp
public override void OnMissionBehaviorInitialize()
{
    base.OnMissionBehaviorInitialize();
    if (Agent.Main != null)
    {
        TryInitializeUI();  // 可能永远不会触发
    }
}

public override void OnAgentCreated(Agent agent)
{
    if (agent == Agent.Main)
    {
        TryInitializeUI();  // 如果Agent已创建，不会触发
    }
}
```

**正确写法**:
```csharp
// 在OnMissionTick中每帧检查，彻底规避生命周期问题
public override void OnMissionTick(float dt)
{
    base.OnMissionTick(dt);
    
    // 懒初始化：Agent.Main存在且未初始化时创建UI
    if (!_triedInit && Agent.Main != null && MissionScreen != null)
    {
        _triedInit = true;
        TryInitializeUI();
    }
    
    // 更新数据...
}
```

**经验教训**: 
- MissionView/MissionBehavior的生命周期回调调用时机不可靠
- 最稳妥的方式是在OnMissionTick中懒初始化
- 或者用_timeout机制，几秒后强制初始化

### 返工6：GauntletLayer构造函数参数错误

**现象**: 编译错误 CS7036 - 未提供与'GauntletLayer.GauntletLayer(string, int, bool)'的所需参数'localOrder'对应的参数

**原因**: 
- 误以为GauntletLayer有只接受int参数的构造函数
- 实际上GauntletLayer的构造函数需要3个参数：layerName、globalOrder、localOrder

**错误代码**:
```csharp
_gauntletLayer = new GauntletLayer(30)  // 错误！参数不对
{
    IsFocusLayer = false
};
```

**正确写法**:
```csharp
_gauntletLayer = new GauntletLayer("NewZZZF_MissionHUD", 30, false)
{
    IsFocusLayer = false
};
```

**参数说明**:
- `layerName`: Layer的名称，用于标识
- `globalOrder`: 全局渲染顺序，数字越大越靠前
- `localOrder`: 是否使用局部排序（一般为false）

**经验教训**: 
- GauntletLayer的构造函数参数要查清楚
- 可以反编译游戏DLL查看正确的签名

### 返工7：XML根Widget缺少Name属性

**现象**: LoadMovie返回null，UI无法加载

**原因**: 
- GauntletUI加载Movie时，需要XML根Widget有Name属性
- Name属性的值就是LoadMovie的第一个参数
- 如果根Widget没有Name，LoadMovie会失败

**错误XML**:
```xml

  <Window>
    <Widget WidthSizePolicy="Fixed" ...>  <!-- 缺少Name属性 -->
      ...
    </Widget>
  </Window>
</Prefab>
```

**正确XML**:
```xml
<Prefab>
  <Window>
    <Widget Name="NewZZZF_MissionAgentStatus" WidthSizePolicy="Fixed" ...>
      ...
    </Widget>
  </Window>
</Prefab>
```

**LoadMovie调用**:
```csharp
// 第一个参数必须是根Widget的Name属性值
_movie = _gauntletLayer.LoadMovie("NewZZZF_MissionAgentStatus", _dataSource);
```

**经验教训**: 
- XML根Widget必须有Name属性
- LoadMovie的第一个参数必须匹配Name属性
- 建议Name属性和文件名保持一致（不含扩展名）

### 返工8：FillBarWidget填充子Widget宽度策略错误

**现象**: 进度条填充不正确，始终显示满值或空值

**原因**: 
- FillBarWidget通过ClipContents裁剪子Widget来显示进度
- 子Widget的宽度应该是StretchToParent，让父Widget控制裁剪
- 如果子Widget宽度固定为220，裁剪就不起作用

**错误XML**:
```xml
<FillBarWidget FillWidget="ManaFill" InitialAmountAsFloat="@ManaCurrent" MaxAmountAsFloat="@ManaMax">
  <Children>
    <Widget Id="ManaFill" WidthSizePolicy="Fixed" SuggestedWidth="220" ... />  <!-- 错误！ -->
  </Children>
</FillBarWidget>
```

**正确XML**:
```xml
<FillBarWidget FillWidget="ManaFill" InitialAmountAsFloat="@ManaCurrent" MaxAmountAsFloat="@ManaMax">
  <Children>
    <Widget Id="ManaFill" WidthSizePolicy="StretchToParent" HeightSizePolicy="StretchToParent" ... />  <!-- 正确 -->
  </Children>
</FillBarWidget>
```

**FillBarWidget工作原理**:
1. FillBarWidget根据InitialAmountAsFloat/MaxAmountAsFloat计算填充比例
2. 按比例裁剪子Widget的显示区域
3. 子Widget应该是StretchToParent，这样裁剪才有效
4. 如果子Widget宽度固定，裁剪区域可能超出或不足

**经验教训**: 
- FillBarWidget的子Widget必须用StretchToParent
- 理解GauntletUI的裁剪机制：ClipContents="true"的Widget会裁剪子Widget
- 进度条的颜色、精灵等可以在子Widget中设置

### 返工9：误将 MissionView 改为 MissionBehavior（重大返工）

**现象**: 编译错误 CS0534 - 不实现继承的抽象成员 `MissionBehavior.BehaviorType.get`

**原因**: 
- AI 错误地将基类从 `MissionView` 改成了 `MissionBehavior`
- `MissionBehavior` 有抽象属性 `BehaviorType` 必须实现
- `MissionView` 才是正确的基类（自带 `MissionScreen` 属性，`OnMissionScreenTick` 回调）
- 用户多次纠正，AI 仍然反复改错

**错误代码**:
```csharp
// AI 错误修改的版本
public class NewZZZF_MissionAgentStatusView : MissionBehavior  // 错误！
{
    public override MissionBehaviorType BehaviorType => MissionBehaviorType.Other; // 不该需要这个
    
    // MissionBehavior 的回调，但 MissionView 用的是 OnMissionScreenTick
    public override void OnMissionTick(float dt)  // 错误回调名！
    {
        // ...
        base.OnMissionTick(dt);  // 错误！
    }
    
    // MissionBehavior 没有 MissionScreen 属性，需要手动获取
```

**正确写法**:
```csharp

{
    // 不需要 BehaviorType
    
    // MissionView 的回调
    public override void OnMissionScreenTick(float dt)  // 正确回调名！
```

**MissionView vs MissionBehavior 对比**:

| 特性 | MissionView | MissionBehavior |

**经验教训**: 
- 用户写的代码 `: MissionView` 是正确的，不要自作主张改成 `MissionBehavior`
- 看到编译错误 CS0534（不实现抽象成员），先检查是不是基类改错了
- `MissionView` 和 `MissionBehavior` 的回调名不同，不能混用

### View (NewZZZF_MissionAgentStatusView.cs)

```csharp
public class NewZZZF_MissionAgentStatusView : MissionView  // 继承MissionView（UI视图的正确基类）
{
    private GauntletLayer _gauntletLayer;
    private NewZZZF_MissionAgentStatusVM _dataSource;
    private GauntletMovieIdentifier _movie;
    private bool _initialized;
    private bool _triedInit;

    // 关键：在OnMissionScreenTick中懒初始化，规避生命周期问题
    public override void OnMissionScreenTick(float dt)
    {
        base.OnMissionScreenTick(dt);

        // 懒初始化（MissionView自带MissionScreen属性，直接用this.MissionScreen）
        if (!_triedInit && Agent.Main != null && this.MissionScreen != null)
        {
            _triedInit = true;
            TryInitializeUI();
        }

        // 更新数据
        if (_dataSource != null && Agent.Main != null)
        {
            SkillSystemBehavior.ActiveComponents.TryGetValue(Agent.Main.Index, out var comp);
            _dataSource.UpdateFromComponent(comp);
        }
    }

    private void TryInitializeUI()
    {
        if (_initialized) return;
        _initialized = true;

        try
        {
            _dataSource = new NewZZZF_MissionAgentStatusVM();

            // 正确的GauntletLayer构造函数
            _gauntletLayer = new GauntletLayer("NewZZZF_MissionHUD", 30, false)
            {
                IsFocusLayer = false
            };

            // LoadMovie第一个参数 = XML根Widget的Name属性
            _movie = _gauntletLayer.LoadMovie("NewZZZF_MissionAgentStatus", _dataSource);

            if (_movie == null)
            {
                // 处理加载失败
                return;
            }

            this.MissionScreen.AddLayer(_gauntletLayer);  // MissionView自带MissionScreen
        }
        catch (Exception ex)
        {
            // 异常处理
        }
    }

    public override void OnRemoveBehavior()
    {
        // 清理Layer和Movie
        if (_gauntletLayer != null && _movie != null)
        {
            _gauntletLayer.ReleaseMovie(_movie);
            _movie = null;
        }
        if (_gauntletLayer != null)
        {
            if (this.MissionScreen != null)
                this.MissionScreen.RemoveLayer(_gauntletLayer);
            _gauntletLayer = null;
        }
        _dataSource = null;
        base.OnRemoveBehavior();
    }
}
```

### 2. 生命周期回调的不可靠性

**问题**: MissionView/MissionBehavior的生命周期回调调用时机不可靠

**解决方案**:
- 不要在生命周期回调中做关键初始化
- 改用OnMissionTick懒初始化
- 或者用超时机制（几秒后强制初始化）

**推荐模式**:
```csharp
private float _initTimeout = 5f;  // 5秒超时

public override void OnMissionTick(float dt)
{
    base.OnMissionTick(dt);
    
    if (!_initialized)
    {
        _initTimeout -= dt;
        if (Agent.Main != null || _initTimeout <= 0)
        {
            TryInitializeUI();
        }
    }
}
```

### 3. GauntletUI的调试技巧

**问题**: UI不显示，但没有任何错误提示

**调试步骤**:
1. 检查XML文件是否存在于正确位置（GUI/Prefabs/）
2. 检查XML根Widget是否有Name属性
3. 检查LoadMovie的参数是否匹配Name属性
4. 检查AddLayer是否被调用
