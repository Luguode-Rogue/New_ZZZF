# 2026-06-13 词缀显示修复：多词缀全显示 + UI层Modifier桥接

## 任务描述
修复词缀系统两个显示问题：
1. 装备有多个前后缀时，只显示第一个
2. 同底材装备在UI上显示相同前后缀（模板回退）

## 修改的文件

### 1. `EquipmentAffixSystem/Data/AffixInstance.cs` — BuildFullName 全显示

**改动**：第102-121行，将 `prefixes[0]` / `suffixes[0]` 改为 `string.Join(" ", ...)` 拼接全部词缀。

```csharp
// 旧：取第一个
string prefixName = hasPrefix ? prefixes[0].DisplayName : null;
string suffixName = hasSuffix ? suffixes[0].DisplayName : null;

// 新：拼接全部
string prefixName = hasPrefix ? string.Join(" ", prefixes.Select(p => p.DisplayName)) : null;
string suffixName = hasSuffix ? string.Join(" ", suffixes.Select(p => p.DisplayName)) : null;
```

同时保留第一个前缀/后缀用于构建 "X之Y的Z" 格式的 Stem。

### 2. `EquipmentAffixSystem/Campaign/AffixCampaignBehavior.cs` — Modifier桥接基础设施

**新增字段**：
- `ModifierToInstanceMap: Dictionary<string, string>` — ItemModifier.StringId → InstanceId
- `_serializedModifierData` [SaveableField(4)] — 存档序列化

**新增方法**：
- `RegisterModifierBinding(modifierStringId, instanceId)` — 注册映射
- `GetAffixForEquipmentElement(EquipmentElement)` — 统一查询接口：
  1. 优先走 ItemModifier.StringId → ModifierToInstanceMap → 精确实例查找
  2. 无修饰符时回退模板查找

**存档更新**：SyncData 中新增 ModifierMap 的序列化/反序列化。

### 3. `EquipmentAffixSystem/UIPatches/AffixUIPatches.cs` — UI层使用新接口

**改动**：两处 Harmony Patch 从 `GetAffixByBaseItemId(item.StringId)` 改为 `GetAffixForEquipmentElement(element)`，传入完整 EquipmentElement 而非仅 ItemObject。
