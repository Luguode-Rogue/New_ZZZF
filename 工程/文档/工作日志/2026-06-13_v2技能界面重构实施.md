## 关键架构变更

### 1. HeroVM 扩展
- 新增 `Hero` / `Character` 公开字段（供熟练度查询）
- 新增 `Subtitle` 属性（等级/兵种类型标注）
- 新增 `CharacterObject` 构造函数（兵种模板）

### 2. SkillSlotVM 扩展
- 新增 `IsActiveSlot`（目录视图打开时高亮当前编辑槽位）
- 新增 `IsEquipped`（= !IsEmpty，供XML显隐绑定，修复 `!@` 否定语法问题）

### 3. 原位目录视图（替代弹窗）
- 点击槽位 → `ActiveSlot = slotVM` + `IsInCatalogView = true`
- 右侧面板原位显示 `CatalogItems`（按槽位类型过滤 + 搜索过滤）
- Enter 选择技能 → `AssignSkillToSlot` + 退出目录
- Esc 退出目录 → 返回技能槽视图
- 目录中 1~8 可切换槽位

### 4. 三模式切换
- `TargetType.PartyMember`（默认）：玩家家族成员
- `TargetType.TroopTemplate`（F12 解锁）：各文化兵种模板
- `TargetType.LordNPC`（F12 解锁）：所有家族领主
- Tab 循环切换，F12 开关调试模式

### 5. 熟练度面板
- 18项骑砍2原生技能（单手/双手/.../工程）
- 通过 `SkillProficiencyVM.GetProficiencies(Hero/BasicCharacterObject)` 获取

## 已规避的已知 Bug

| v2设计文档记录的Bug | 修复方式 |
|---|---|
| 技能槽不自动加载 | `SetSkill()` 完整通知链（6项属性），`LoadSkillsForTarget` 全槽位刷新 |
| UI不刷新 | 所有属性使用 `OnPropertyChangedWithValue`；`!@` 否定→ `IsEquipped`/`IsShowingSlots` 逆属性 |
| 布局问题 | GridWidget/WrapPanel→ListPanel；所有滚动区 ClipContents；AlphaFactor≥0.08 |
| SwitchTargetType 不更新类型 | `_currentTargetType` + `OnPropertyChanged` 提前到方法开头 |
